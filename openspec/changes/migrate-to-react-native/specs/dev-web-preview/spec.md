## 新增需求

### 需求:开发构建必须支持 Expo Web 预览命令

工程必须提供可通过命令行启动的 Web 预览入口（例如 `expo start --web` 或等价 npm script），用于在桌面浏览器中查看主要页面与导航。

#### 场景:启动 Web 预览

- **当** 开发者在项目目录执行 Web 预览脚本
- **那么** 浏览器必须能访问首页与动物列表页的主要 UI 结构

### 需求:Web 预览不得作为生产或家长模式验收环境

文档与团队约定必须明确：Expo Web 预览仅用于开发与 UI 调试，不得作为荣耀未成年人模式、原生 TTS 或最终音频行为的验收依据。

#### 场景:验收标准声明

- **当** 进行发布前验收
- **那么** 必须通过 Android 真机或 release APK 验证播放与家长模式相关行为，不得仅以 Web 预览通过为准

### 需求:Web 预览在音频失败时不得阻塞界面

当 Web 环境下原生音频或 TTS 不可用时，界面必须仍可浏览列表与打开详情；允许播放失败或降级，但必须给出可继续操作的状态（不白屏、不崩溃）。

#### 场景:Web 环境打开详情

- **当** 开发者在浏览器预览中打开动物详情且音频 API 不可用
- **那么** 详情界面必须仍可显示并可关闭返回列表
