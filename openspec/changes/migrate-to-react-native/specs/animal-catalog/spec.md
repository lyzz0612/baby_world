## 新增需求

### 需求:动物列表必须展示全部动物且不得按分类 Tab 筛选

动物列表必须合并展示所有类别动物，禁止提供农场/动物园/海洋馆等分类切换 Tab 或等价筛选 UI。

#### 场景:进入动物列表

- **当** 用户打开动物列表页
- **那么** 界面不得显示分类 Tab，且必须展示当前页的动物卡片集合

### 需求:动物列表必须使用按钮翻页而非滑动手势翻页

系统必须通过左右翻页按钮切换页面，禁止将横滑手势作为翻页的唯一或主要方式。每页必须固定展示 8 个动物槽位（末页不足 8 个时允许留白或减少展示数量）。

#### 场景:点击下一页

- **当** 用户点击「下一页」且存在后续页
- **那么** 列表必须切换到下一页并展示对应的动物卡片

#### 场景:末页禁止下一页

- **当** 用户位于最后一页并点击「下一页」
- **那么** 系统必须保持当前页且不得产生翻页动作（按钮禁用或无效反馈）

#### 场景:首页禁止上一页

- **当** 用户位于第一页并点击「上一页」
- **那么** 系统必须保持当前页且不得产生翻页动作（按钮禁用或无效反馈）

### 需求:动物卡片必须可点击进入详情播放流程

每个动物卡片必须展示 emoji 与名称，且点击卡片必须触发该动物的详情/播放流程。

#### 场景:点击动物卡片

- **当** 用户点击某一动物卡片
- **那么** 系统必须打开该动物的详情/播放界面（Modal 或全屏页）

### 需求:列表必须支持返回首页

动物列表必须提供返回首页的控件。

#### 场景:返回首页

- **当** 用户点击返回控件
- **那么** 系统必须导航回首页
