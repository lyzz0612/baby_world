## 新增需求

### 需求:系统必须记录每个动物的点击次数

每次用户点击动物卡片进入详情时，系统必须将该动物的累计点击次数加 1，并持久化到本地存储。

#### 场景:点击动物累计次数

- **当** 用户点击动物 A 的卡片进入详情
- **那么** 动物 A 的本地累计点击次数必须比之前增加 1

#### 场景:应用重启后保留统计

- **当** 用户关闭并重新打开应用
- **那么** 各动物的累计点击次数必须与关闭前一致

### 需求:动物列表必须按点击次数降序排列

动物列表的全局顺序必须按点击次数从高到低排序；点击次数相同时，必须保持数据文件中的原始顺序。

#### 场景:高频动物排在前面

- **当** 动物 A 的点击次数大于动物 B
- **那么** 在列表数据中动物 A 必须排在动物 B 之前

#### 场景:点击次数相同保持原始顺序

- **当** 动物 A 与动物 B 点击次数相同且 A 在原始数据中先于 B
- **那么** 排序后 A 仍必须排在 B 之前

### 需求:关闭详情后必须刷新排序

当用户关闭动物详情界面后，系统必须根据最新点击统计重新排序列表；若当前页码超出新总页数，必须自动调整到有效页码。

#### 场景:关闭详情后热度上升

- **当** 用户多次点击动物 A 使其成为最高热度并关闭详情
- **那么** 动物 A 必须出现在更靠前的页码中（通常为第一页）
