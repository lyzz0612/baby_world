## 新增需求

### 需求:RN 应用必须使用原生 UI 壳而非 WebView 主界面

移动客户端必须以 React Native 原生组件渲染主界面，禁止以 WebView 或远程 URL 作为主 UI 容器。Android 应用包名必须为 `com.animal.app`，应用显示名称必须为「动物乐园」。

#### 场景:Android 启动应用

- **当** 用户在 Android 设备上打开已安装 APK
- **那么** 系统展示由 RN 原生视图构成的首页，且不加载外部 http(s) 网页作为主界面

#### 场景:应用元数据

- **当** 查看系统设置或启动器中的应用信息
- **那么** 应用名显示为「动物乐园」，包名为 `com.animal.app`

### 需求:应用必须提供首页与动物模块入口

系统必须提供首页，且首页必须包含进入动物列表的明确入口。路由必须至少包含根路径（首页）与动物列表路径。

#### 场景:从首页进入动物列表

- **当** 用户在首页点击「认识动物」或等价主入口
- **那么** 系统导航至动物列表界面

### 需求:Android 离线运行不得依赖公网

生产构建的 Android 应用必须在没有互联网连接时仍可进入首页并完成动物浏览与本地音频播放（在资源文件存在的前提下）。

#### 场景:飞行模式下使用

- **当** 设备处于无网络状态且用户打开应用
- **那么** 应用必须能正常进入动物列表并响应点击（不因网络不可用而崩溃或空白屏）
