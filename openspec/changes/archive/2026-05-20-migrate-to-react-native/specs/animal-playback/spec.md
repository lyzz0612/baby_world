## 新增需求

### 需求:打开动物详情时必须自动开始播放

当用户进入某一动物的详情/播放界面时，系统必须自动开始该动物的播放序列，无需用户再次点击播放。

#### 场景:进入详情自动播放

- **当** 用户打开动物 A 的详情界面
- **那么** 系统必须立即开始动物 A 的播放序列

### 需求:播放序列必须先 TTS 再动物叫声

播放序列必须先播放该动物的「怎么叫」语音提示，再播放真实动物叫声（若资源存在）。语音提示优先使用预打包的 TTS mp3 文件。

#### 场景:存在预生成 TTS mp3

- **当** 动物 A 存在对应的 `tts/{id}.mp3` 资源且用户触发播放
- **那么** 系统必须先播放该 mp3，并在其结束后再播放动物 A 的叫声文件（若存在）

#### 场景:TTS mp3 缺失时必须兜底

- **当** 动物 A 的预生成 TTS mp3 不存在或播放失败
- **那么** 系统必须使用系统 TTS 朗读「{动物名}怎么叫」（语言 zh-CN），然后继续播放叫声文件（若存在）

#### 场景:仅有叫声无 TTS 文件

- **当** TTS 兜底也失败但动物 A 存在叫声文件
- **那么** 系统必须至少播放动物 A 的叫声文件

### 需求:用户必须能手动重播

详情界面在播放结束后必须提供「再听一次」或等价控件，点击后必须重新执行完整播放序列。

#### 场景:重播

- **当** 用户在详情界面点击重播控件
- **那么** 系统必须停止当前播放并重新开始该动物的完整播放序列

### 需求:关闭详情时必须停止音频

当用户关闭详情界面时，系统必须立即停止所有正在进行的音频与 TTS。

#### 场景:关闭详情

- **当** 用户关闭动物 A 的详情界面
- **那么** 系统必须停止所有音频/TTS 且不得继续播放动物 A 的内容

### 需求:切换动物时必须取消上一轮播放

当用户在详情打开状态下切换到另一动物，或关闭后打开另一动物时，系统必须取消上一轮未完成的播放，仅保留最新一次播放请求的结果。

#### 场景:快速切换动物

- **当** 用户连续打开动物 A 与动物 B 的详情
- **那么** 系统不得让动物 A 的播放在动物 B 开始后仍改变 B 的播放状态
